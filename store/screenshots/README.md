# ストア用スクリーンショット作成ガイド

App Store / Google Play 申請に必要なスクリーンショットの仕様と作成手順をまとめます。

ディレクトリ構成（推奨）:

```
store/screenshots/
├── ios/
│   ├── 6.7/        # iPhone 6.7" (必須)
│   ├── 6.5/        # iPhone 6.5"
│   └── 12.9/       # iPad Pro 12.9" (タブレット対応申請時に必須)
├── android/
│   ├── phone/      # スマートフォン (必須)
│   ├── tablet-7/   # 7インチタブレット
│   └── tablet-10/  # 10インチタブレット
└── source/         # 元データ (Figma 等) があれば
```

---

## iOS（App Store Connect）

App Store Connect では、最大サイズのデバイス1種類分を用意すれば、他サイズは自動でリサイズ表示されます。**最低限 6.7" を用意**してください。

| デバイス | 解像度 (px, 縦) | 必須 | 備考 |
| --- | --- | --- | --- |
| iPhone 6.7" (15 Pro Max 等) | 1290 × 2796 | ◯ | 最優先で用意 |
| iPhone 6.5" (11 Pro Max 等) | 1242 × 2688 | △ | 任意（自動リサイズ可） |
| iPad Pro 12.9" (第2〜6世代) | 2048 × 2732 | △ | `supportsTablet: true` のため用意推奨 |

- 形式: PNG または JPEG（RGB、アルファチャンネルなし）
- 枚数: 1〜10枚（各デバイスサイズごと）
- 向き: ポートレート（本アプリは縦固定）

---

## Android（Google Play Console）

| 種別 | 推奨解像度 (px) | 必須 | 備考 |
| --- | --- | --- | --- |
| スマートフォン | 1080 × 1920 以上 | ◯ | 最低2枚 |
| 7インチタブレット | 1200 × 1920 程度 | △ | タブレット対応時 |
| 10インチタブレット | 1600 × 2560 程度 | △ | タブレット対応時 |

- 形式: PNG または JPEG（24bit、アルファなし）
- 縦横比: 16:9 〜 9:16 の範囲
- 枚数: 各種別 2〜8枚
- 別途必要な素材:
  - アプリアイコン: 512 × 512 px (PNG, 32bit)
  - フィーチャーグラフィック: 1024 × 500 px (PNG/JPEG)

## 撮影手順

1. ビルドを起動（シミュレータ/実機）。
   ```bash
   # biz
   npm run start:biz
   # consumer
   npm run start:consumer
   ```
2. 各画面（ホーム / 記録一覧 / グラフ / エクスポート / 設定）を表示。
3. 端末またはシミュレータでスクリーンショットを取得。
   - iOS Simulator: `Cmd + S`（デスクトップに保存）
   - Android Emulator: ツールバーのカメラアイコン
4. 上記ディレクトリにデバイス別に保存。
5. 必要に応じてデバイスフレーム・キャッチコピーを合成（任意）。

## 推奨する掲載画面（共通の流れ）

1. ホーム（現在のWBGTと警戒レベル）
2. リアルタイムアラート表示
3. 記録一覧（時系列）
4. グラフ / 推移
5. CSV・PDF エクスポート（biz）/ 活動記録（consumer）

> flavor（biz / consumer）ごとにアプリ名・配色が異なるため、**それぞれ別々に撮影**してください。
